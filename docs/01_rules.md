# Rules and guidelines

## Who can participate

Solo or teams of up to 5. Any skill level, anywhere. You must be 18+ or have parental consent.

## Registration and teams

1. Join Discord/Telegram.
2. Register during the hackathon window (March 23 – April 27, 2026).
3. Form a team if you want; solo is fine. No track to pick.
4. Start building.

Max 5 per team. Everyone on the team should register. You can form the team before or after the hackathon starts.

## Development

**Code:** Build during the 5 weeks. You can use open source libs and frameworks. Building on your own prior work is OK if the new part is substantial. Starter templates are there to use.

**Tech:** MIT, Apache 2.0, or GPL v3. Repo must be public on GitHub. README with setup and how to run. Project has to run and do what you demo. It has to use Conflux in a real way, not just in name.

**AI tools:** Copilot, ChatGPT, Cursor, etc. are allowed. What you submit should still be your team’s work.

## Timeline

5 weeks: March 23 – April 27, 2026. Registration same window. Everything due April 27, 11:59:59 UTC. No extensions. Judging takes 3 weeks; no live rounds. Winners announced May 18. Winners send short videos for the winners reel.

## What you must submit

- Public GitHub repo with source code
- Your project under `/projects/` in the hackathon repo
- A 3–5 minute demo video
- A **participant intro video** (30–60 seconds): front-facing, clean audio, simple background. Say who you are, what you're building for Global Hackfest 2026, and that you're excited to participate. These videos will be used at a conference in Hong Kong.
- README plus project docs, including a go-to-market plan
- GitHub issue filled out with the submission template
- A PR to [electric-capital/open-dev-data](https://github.com/electric-capital/open-dev-data) adding your project
- A tweet (or equivalent) tagging @ConfluxDevs and @ConfluxNetwork

Optional: Conflux Forum grant proposal (+5 pts), live deployed app, slides, extra docs.

Submissions missing any required item (other than explicitly optional ones) may be disqualified from prize eligibility.

We expect it to run, code to be readable, and docs to be enough for someone else to run it. Conflux should be used in a real way.

## IP and licensing

You keep the copyright. We need an open license (MIT, Apache 2.0, or GPL v3). Conflux gets a non-exclusive right to use and show your project. Credit your team and any third-party or AI help in the docs.

## Judging

We score: technical (25), Conflux use (25), UX (20), innovation (20), presentation/docs (10). We select 5 Main Award winners and 5 Category Award winners (Best USDT0 integration, Best AxCNH integration, Best AI + Conflux, Best DeFi project, Best Developer Tool). See the judging criteria doc for details.

## Prizes

5 Main Awards ($1,500 each), 5 Category Awards ($500 each): Best USDT0 integration, Best AxCNH integration, Best AI + Conflux, Best DeFi project, Best Developer Tool. Total $10K. Submitting an optional Conflux Forum grant proposal can earn +5 bonus points in judging. Partner or sponsor prizes are run by them; Conflux isn’t responsible for their choices or payouts. If you go for a partner bounty, read that bounty’s rules. Payout for our prizes is in CFX on eSpace. You handle taxes; we may ask for KYC for larger amounts. You have to send the winner video before we send the prize (see submission guide).

## Community

Be respectful. Use Telegram and Discord for tech questions, GitHub for submissions. No plagiarism; credit others’ code. Don’t harass or spam. We follow the Code of Conduct; violations can get you warned, disqualified, or banned.

By taking part you agree to these rules.
