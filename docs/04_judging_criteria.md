# Judging criteria

We score projects on five areas. No fixed tracks; we look at the whole thing.

## 1. Technical quality (25 pts)

Code, structure, how it’s built, and how well it’s documented.

| Score | What it means |
|-------|----------------|
| 20–25 | Strong code, clear architecture, good error handling and docs |
| 15–19 | Solid code and structure, decent docs |
| 10–14 | Works, structure is OK, docs are minimal |
| 5–9 | Messy or buggy, weak or missing docs |
| 0–4 | Doesn’t run or has no real structure/docs |

We care about: how it’s organized, patterns and practices, performance and security where relevant, tests, and whether someone else could run and extend it.

## 2. Conflux integration (25 pts)

Whether and how you use Conflux, not just “any chain.”

| Score | What it means |
|-------|----------------|
| 20–25 | Excellent use of Conflux |
| 15–19 | Good use of Conflux |
| 10–14 | Basic Conflux use |
| 5–9 | Barely Conflux-specific |
| 0–4 | No real Conflux use |

## 3. User experience (20 pts)

Design, usability, and whether it’s actually pleasant to use.

| Score | What it means |
|-------|----------------|
| 16–20 | Clear, intuitive, works on mobile, accessible |
| 12–15 | Good enough; a few rough edges |
| 8–11 | Usable but clunky |
| 4–7 | Confusing or frustrating |
| 0–3 | Broken or unusable |

We look at layout, navigation, accessibility, load times, errors and feedback, and onboarding.

## 4. Innovation (20 pts)

How fresh the idea is and whether it could matter.

| Score | What it means |
|-------|----------------|
| 16–20 | New angle, clear problem, real upside |
| 12–15 | Interesting take, some novelty |
| 8–11 | Small twist on something known |
| 4–7 | Mostly familiar |
| 0–3 | Copy of something that already exists |

We consider: novelty, how you framed the problem, possible impact, and any technical or product twist.

## 5. Presentation and docs (10 pts)

Demo video and written explanation (including go-to-market).

| Score | What it means |
|-------|----------------|
| 8–10 | Clear demo, easy to follow, good docs |
| 6–7 | Understandable; a bit rough |
| 4–5 | We get the gist |
| 2–3 | Unclear or messy |
| 0–1 | No usable demo or explanation |

Demo length 3–5 min. All submissions must also include a participant intro video (30–60 sec). After we announce winners, they send an additional short pitch clip for the winners reel.

## Bonus

+5 pts if you submit a grant proposal to the [Conflux Forum](https://forum.conflux.fun/c/English/grant-proposals) with a concrete post-hackathon plan. Use the [Integration Grants Application Template](https://forum.conflux.fun/t/integration-grants-application-template/20759) when drafting it.
